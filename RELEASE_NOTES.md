# 🚀 Release v5.0.1 | 26.03.2026: Стабилизация и исправление критических ошибок

Релиз направлен на устранение критических ошибок совместимости между современным рантаймом Node.js 22 в Yandex Cloud и устаревшим бандлом воркера. Основное достижение — восстановление бинарного транспорта для Telegram API и внешних конвертеров.

## 🐞 Исправленные критические ошибки:

*   **Telegram Error:** Невозможность редактировать медиа-сообщения (фото/видео) после обработки.
*   **Runtime Error:** Падение функции при попытке вызвать методы событий на бинарных данных (`TypeError: source.on is not a function`).
*   **Logic Error:** Постоянные циклы `[USER-UPDATE]` из-за конфликтов в обращении к БД (частично решено стабилизацией окружения).
*   **Multipart Error:** Решена проблема `parameter "media" is required` из-за неправильного `boundary` в multipart-запросах.

## 🛠 Архитектурные изменения:

*   **Yandex.Cloud Gateway:** Организован отдельный шлюз для обеспечения совместимости и маршрутизации запросов, что позволяет не переделывать эндпоинты в коде воркера.

*   **Прокси-слой `index.js`:** `index.js` теперь выступает как слой обратной совместимости. 
    *   **Схема обработки запроса:**
        1.  **Перехват:** Все вызовы `fetch` проходят через `smartFetch`.
        2.  **Детекция:** Если в теле запроса обнаружен `FormData`, включается режим пересборки.
        3.  **Трансформация:** Данные переводятся в «безопасный» `Buffer` с методами-заглушками.
        4.  **Доставка:** Запрос уходит через `originalFetch` (нативный fetch Node.js), гарантируя максимальную скорость и поддержку TLS.

*   **Manual Multipart Assembly (Ручная сборка):** Внедрена функция `prepareMultipart`, которая заменяет стандартные методы `form-data`, решая проблемы с `boundary` и `Content-Length`.

*   **Stream API Emulation (Эмуляция потоков):** Для объектов `Buffer` добавлены методы-заглушки (`.on()`, `.pause()`, `.resume()`), чтобы «обмануть» старые библиотеки, ожидающие Stream.

*   **Dependency Hijacking (Перехват зависимостей):** Реализована подмена модулей через `require.cache` для `node-fetch` и `form-data`, что позволило внедрить исправления в закрытый скомпилированный код воркера.

## ❗ Важное примечание по YDB:
> При получении ошибки `RESOURCE_EXHAUSTED` (код 8), рекомендуется сделать паузу в запросах на 5-10 минут. Это связано с лимитами Serverless-тарифного плана на количество операций (RU) при частых перезагрузках функции.

## 🚀 Ключевое изменение v5.X: Переезд на Yandex.Cloud
- **Среда выполнения**: Бот теперь работает в полноценной среде Node.js 22, что обеспечивает стабильность и расширяемость.
- **База данных**: В качестве основного хранилища данных (вместо Cloudflare KV) используется **Yandex Database (YDB)**.
- **Архитектура**: Разработан специальный адаптер (`index.js` + `db_adapter.js`), который эмулирует среду Cloudflare Workers, позволяя сохранить и адаптировать основную кодовую базу `worker.js`.
