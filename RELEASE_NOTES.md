# 🚀 Release v5.0 | 24.03.2026: Миграция на Yandex.Cloud и расширение функционала

## 🚀 Ключевое изменение: Переезд на Yandex.Cloud
Этот релиз знаменует полный переезд бота с платформы Cloudflare Workers на **Yandex.Cloud Functions**. Это изменение позволило преодолеть технические ограничения предыдущей платформы и открыло новые возможности для развития.
- **Среда выполнения**: Бот теперь работает в полноценной среде Node.js 22, что обеспечивает стабильность и расширяемость.
- **База данных**: В качестве основного хранилища данных (вместо Cloudflare KV) используется **Yandex Database (YDB)**. Это обеспечивает надежное хранение пользовательских данных, истории чатов и медиафайлов.
- **Архитектура**: Разработан специальный адаптер (`index.js` + `db_adapter.js`), который эмулирует среду Cloudflare Workers, позволяя сохранить и адаптировать основную кодовую базу `worker.js`.

## ✨ Новые возможности и улучшения
- **Полифиллы и совместимость**: В `index.js` добавлены полифиллы для `atob` и `btoa` для обеспечения совместимости кода в среде Node.js.
- **Единое хранилище**: Все обращения к различным KV-хранилищам (`LAST_PHOTO_STORAGE`, `CHAT_HISTORY_STORAGE` и др.) теперь централизованно обрабатываются через единый адаптер `USER_DB_ADAPTER`, работающий с YDB.
- **Работа с метаданными**: В `db_adapter.js` добавлен метод `getWithMetadata` для корректной обработки медиафайлов, требующих метаданные.
- **Экспорт модуля**: В `worker.js` добавлена строка `module.exports = { worker_code_fetch };` для обеспечения возможности вызова кода из среды Node.js.
